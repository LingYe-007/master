## 一、风格参考（图12：属性语义图构建流程）

图12的核心风格特征：
- **布局**：横向流程，三大块（A/B/C）从左到右，用虚线分隔
- **形状**：矩形框表示过程/数据，椭圆框表示算法/概念
- **配色**：白底、浅蓝/浅橙/浅绿区分模块，公式框为灰白
- **线条**：折线箭头，无曲线，直角转折
- **文字**：中英双语标签，公式清晰排版
- **装饰**：云形标注说明特殊条件，小图标表示概念（用户、物品等）

## 二、主 Prompt（用于 AI 绘图 / 设计师）

```
请以「属性语义图构建流程示意图」的绘图风格，重新绘制「FedASCL算法总体架构示意图」。

【风格要求】
- 布局：横向流程图，分区清晰，用虚线框划分服务端、客户端
- 形状：过程用矩形，算法/聚合模块用圆角矩形或椭圆
- 线条：全部为折线箭头（直角转折），禁止曲线
- 配色：白底，浅灰分区背景，蓝/橙区分不同模块，与参考图一致
- 文字：中英双语，公式简洁

【内容要求 - 六步流程】
1. 双视图构建：用户/物品原始属性 → 属性语义图；本地交互数据 → 交互结构图
2. 双视图编码：两个 GNN 编码器 → 语义节点表示、结构节点表示
3. 跨视图对比：两表示 → 对比学习模块（最大化互信息）
4. 原型对齐：结构表示 + 全局原型下传 → 原型对齐模块
5. 模型聚合与更新：服务端两模块（全局语义原型聚合、模型聚合）
6. 联合优化与推荐：元路径挖掘 → 推荐预测 → 损失计算 → 联合优化

【端云箭头】
- Global Prototypes：服务端蓝云 → 客户端原型对齐
- Updated Model Parameters：客户端联合优化 → 服务端橙云
- Local Prototypes：原型对齐 → 服务端蓝云（虚线）
- Global Model Update：服务端橙云 → 客户端原始属性输入（虚线）

【图标】
- 服务端：蓝云（原型聚合）、橙云+齿轮（模型聚合）
- 客户端：六边形表示属性语义图、交互结构图，放大镜表示对比学习
```

## 三、分步 Prompt（用于分块绘制）

### 1. 布局与分区
```
绘制 FedASCL 架构图，采用与「属性语义图构建」相同的版面：
- 顶部：服务端区域（浅灰背景，虚线框），内含蓝云、橙云两模块
- 中部：客户端主流程（虚线框「Iterative Local Training」）
- 左右横向流动，模块间用折线箭头连接
```

### 2. 模块与样式
```
- 输入/数据框：灰色圆角矩形
- 算法框（GNN、对比、原型对齐）：白色矩形，带边框
- 图结构模块：六边形，蓝/橙填充
- 服务端云形：多圆叠加的云形，蓝/橙配色
- 模型聚合：橙云 + 内部齿轮图标
```

### 3. 线条与箭头
```
- 所有连接线：折线（polyline），直角转折
- 实线：主要数据流
- 虚线：原型上传、Global Model 下传
- 箭头：标准实心三角
```

## 四、Python/Matplotlib 实现提示

若要由 `draw_fedascl_framework.py` 实现图12风格，可调整：
- `figsize` 改为更宽的横向比例（如 16×8）
- 增加分区虚线框 `FancyBboxPatch(..., linestyle='--')`
- 模块用 `round,pad` 圆角，算法模块可改用椭圆 `Ellipse`
- 统一 `polyline_arrow` 折线，禁止 `arc3` 曲线
- 配色与图12一致：`#e3f2fd`（蓝）、`#fff3e0`（橙）、`#e8f5e9`（绿）
