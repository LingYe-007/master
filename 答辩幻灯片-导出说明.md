## 一、Cursor / VS Code 插件（图形界面导出）

- **扩展名称**：**Marp for VS Code**  
- **扩展 ID**：`marp-team.marp-vscode`  
- **安装方式**（任选其一）：  
  1. 在 Cursor 中 `Cmd+Shift+X` → 搜索 **Marp** → 安装 **Marp for VS Code**。  
  2. 终端执行：`cursor --install-extension marp-team.marp-vscode`（若已安装会提示已存在）。

**导出步骤**：用 Cursor 打开 `答辩幻灯片.md` → 右上角或命令面板选择 **Marp: Export Slide Deck** → 选 **PowerPoint (.pptx)** 或 **PDF**。

## 二、命令行导出（不依赖编辑器）

本机需已安装 **Node.js**（你当前环境在 `~/.nvm` 下为 v22.18.0）。在论文根目录执行：

```bash
export PATH="$HOME/.nvm/versions/node/v22.18.0/bin:$PATH"
cd /Users/wxg/Projects/毕业论文
npx --yes @marp-team/marp-cli@latest ./答辩幻灯片.md -o ./output/伍勋高-答辩.pptx --pptx --no-stdin --allow-local-files
```

- `--pptx`：输出 PowerPoint。  
- `--no-stdin`：避免 CLI 等待标准输入。  
- `--allow-local-files`：允许引用 `images/` 等本地路径（否则图片可能被安全策略拦截）。

导出 **PDF** 时把 `--pptx` 去掉并改输出扩展名即可，例如 `-o ./output/伍勋高-答辩.pdf`。

## 三、可选：把 Marp CLI 固定到本仓库（便于团队）

```bash
cd /Users/wxg/Projects/毕业论文
npm init -y
npm install --save-dev @marp-team/marp-cli
```

之后在 `package.json` 里加脚本：

```json
"scripts": {
  "slides:pptx": "marp 答辩幻灯片.md -o output/伍勋高-答辩.pptx --pptx --no-stdin --allow-local-files"
}
```

执行：`npm run slides:pptx`。

## 四、常见问题

1. **终端里 `node: command not found`**：先执行 `export PATH="$HOME/.nvm/versions/node/当前版本/bin:$PATH"`，或把该路径写入 `~/.zshrc`。  
2. **PPTX 里图不显示**：务必加 `--allow-local-files`，且导出时在**论文根目录**执行（相对路径 `./images/...` 才正确）。  
3. **WPS 打开版式略变**：属正常现象，可在 WPS 里统一字体为微软雅黑/宋体后再微调。
