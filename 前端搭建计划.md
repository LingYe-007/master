# 联邦论文推荐系统 — 前端搭建计划

依据论文第五章「基于联邦异构图模型的推荐系统设计与实现」中的业务展示层与用户需求，制定本计划。技术栈与论文一致：**React + TypeScript + ECharts**。

| 学术资源推荐     | 根据是否有交互历史切换「结构优先/属性优先」；论文列表；元路径推荐理由     | **推荐列表页** |

### 1.4 业务逻辑要点（供前端对接）

- **推荐策略**：有交互历史 → 结构优先（结构视图）；无交互 → 属性优先（属性映射 + 全局原型），冷启动。
- **可解释性**：展示元路径注意力权重带来的推荐理由，例如「推荐该学者是因为你们有共同的合作者」。
- **监控指标**：参与客户端数、当前轮次、全局 Loss、AUC 随轮次曲线、各客户端在线状态。

## 二、前端功能范围与页面清单

### 2.1 页面清单

| 序号 | 页面           | 路由（建议）        | 角色     | 说明 |
|------|----------------|---------------------|----------|------|
| 1    | 登录/注册      | `/login`, `/register` | 所有用户 | 联邦身份认证入口 |
| 2    | 推荐首页      | `/` 或 `/recommend`  | 已登录用户 | 学术资源推荐列表，Top-N 论文，推荐理由 |
| 3    | 个人设置/画像  | `/settings`          | 已登录用户 | 研究兴趣、所属机构、研究方向等 |
| 4    | 联邦训练监控   | `/admin/monitor`     | 管理员   | Loss/AUC 曲线、轮次、客户端状态 |

### 2.2 全局与公共部分

- **布局**：顶栏（Logo、导航、用户信息/退出）、侧栏或顶栏导航（推荐、个人设置；管理员可见监控）。
- **路由**：React Router，保护路由（未登录跳登录；管理员页校验角色）。
- **API 层**：统一 baseURL、请求/响应拦截、与后端 RESTful 对接（推荐列表、冷启动、训练状态、用户画像等）。
- **状态**：用户信息与登录态（Context 或 Zustand/Redux 二选一）、可选的主题/语言。

## 三、分步实施计划

### 阶段 0：准备

- **0.1** 确认后端 API 约定（Base URL、登录/注册、推荐、设置、监控接口及数据格式）。若无后端，先使用 Mock 数据与 Mock 服务。
- **0.2** 在项目下创建前端工程目录（如 `fedrec-frontend`），与论文 LaTeX 及 `images` 并列。

### 阶段 1：项目初始化与基础架构

- **1.1** 使用 Vite + React + TypeScript 初始化项目（与论文「React + TypeScript」一致）。
- **1.2** 安装并配置：React Router、ECharts、axios（或 fetch 封装）、UI 组件库（如 Ant Design 或 MUI，便于快速做表单/表格/布局）。
- **1.3** 配置环境变量：`VITE_API_BASE_URL` 等，便于后续对接真实后端。
- **1.4** 实现基础布局组件：顶栏 + 主内容区；导航菜单（推荐、个人设置、管理员监控入口）。
- **1.5** 配置路由：`/login`、`/register`、`/`（或 `/recommend`）、`/settings`、`/admin/monitor`；设置 404 页。

### 阶段 2：登录与注册（联邦身份认证入口）

- **2.1** 登录页：账号/密码（或与后端约定方式），提交后调用登录 API，保存 token 与用户信息，跳转推荐页。
- **2.2** 注册页：用户名、密码、可选邮箱等；提交后调用注册 API，成功后跳转登录或推荐。
- **2.3** 路由守卫：未登录访问 `/`、`/settings` 等重定向到 `/login`；登录后访问 `/login` 可重定向到 `/`。
- **2.4** 顶栏展示当前用户信息及「退出」按钮，退出清除 token 并跳转登录。

### 阶段 3：个人设置页（个人画像管理）

- **3.1** 表单字段与论文一致：**研究兴趣**、**所属机构**、**研究方向**（可多选或标签输入）。
- **3.2** 提交后调用「更新用户属性」API（论文：属性数据存本地，仅加密参与联邦；前端仅负责展示与提交到后端，由后端与客户端逻辑处理）。
- **3.3** 页面说明文案可注明：以上属性用于冷启动阶段属性视图输入，数据仅以加密形式参与联邦训练。

### 阶段 4：学术资源推荐列表页

- **4.1** 列表展示：论文标题、作者、来源（会议/期刊）、摘要或关键词、可选封面图；支持分页或「加载更多」。
- **4.2** 推荐策略展示：在列表或详情处标注当前用户使用的策略（**结构优先** / **属性优先（冷启动）**），与论文 5.3.3 一致。
- **4.3** 推荐理由（可解释性）：每条推荐可展示一句理由，例如「推荐该学者是因为你们有共同的合作者」或基于元路径权重的简短文案；数据来自后端「推荐理由」或「元路径权重」接口。
- **4.4** 若后端未就绪，使用 Mock 列表与 Mock 理由，结构按接口约定预留。

### 阶段 5：联邦训练监控面板（管理员）

- **5.1** 仅管理员可访问 `/admin/monitor`；顶栏或路由中根据角色显示「监控」入口。
- **5.2** 展示内容与论文一致：
  - 参与训练的**客户端数量**、**当前训练轮次**；
  - **全局 Loss**、**AUC** 随轮次变化的曲线（ECharts 折线图）；
  - **各客户端在线状态**（列表或状态标识：在线/离线/训练中）。
- **5.3** 数据由「训练状态」类 API 提供（轮次、loss 序列、auc 序列、客户端列表及状态）；若暂无后端，使用 Mock 数据与定时刷新模拟曲线更新。
- **5.4** 曲线随轮次动态更新（轮询或 WebSocket），正文描述「曲线随训练轮次动态更新且趋势收敛」。

### 阶段 6：联调与收尾

- **6.1** 与后端对接：登录/注册、推荐、个人设置、监控接口，统一错误码与错误提示。
- **6.2** 错误处理：网络错误、401 跳转登录、403 提示无权限。
- **6.3** 简单加载态与空状态：列表加载中、无推荐、无训练任务等。
- **6.4** 可选：将论文中三张界面截图（推荐页、设置页、监控面板）作为 UI 参考或占位，在说明文档中注明与论文图对应关系。

## 五、API 接口约定（与后端对齐）

以下为建议命名与字段，便于前后端联调。

| 功能       | 方法 | 路径（建议）           | 说明 |
|------------|------|------------------------|------|
| 登录       | POST | `/api/auth/login`      | body: { username, password } |
| 注册       | POST | `/api/auth/register`   | body: { username, password, email? } |
| 当前用户   | GET  | `/api/user/me`         | 返回用户信息及是否管理员 |
| 更新画像   | PUT  | `/api/user/profile`    | body: { interests, institution, researchDirections } |
| 推荐列表   | GET  | `/api/recommend`       | 返回论文列表 + 推荐理由/策略标识 |
| 训练状态   | GET  | `/api/admin/training`  | 返回轮次、loss 序列、auc 序列、客户端状态列表 |
