\section{第五章\texorpdfstring{\quad}{} 基于联邦异构图模型的推荐系统设计与实现}
\label{chap:system_design}
\setcounter{section}{5}
\setcounter{subsection}{0}

本章将第三章的 FedASCL 与第四章的语义感知压缩策略集成到面向学术场景的联邦论文推荐原型系统中，用于展示端云协同流程并应对数据异质性与通信瓶颈等工程问题；排序效果以第三章离线实验为准。以下按系统需求分析、总体架构设计、关键模块实现与功能测试四部分展开。

% ----------------------------------------------------------------------
\subsection{系统需求分析}
\label{sec:sys_requirements}

本节从业务与用户两方面对联邦推荐系统做功能定义。

\subsubsection{业务需求分析}

本系统在业务上应支持用户注册并维护本地私有数据库，使用户原始交互始终保留在端侧而不出域，端云之间仅传输经保护的模型参数、梯度或全局语义原型，以落实联邦身份认证并降低隐私泄露风险。训练流程上，服务器向参与客户端下发全局模型与全局原型，客户端在本地完成 FedASCL 双视图对比学习，并依据第四章策略对上传内容进行动态元路径剪枝与残差梯度量化；服务器对收到的更新执行加权聚合，形成闭环的端云协同训练。与此同时，系统应支持可配置的本地差分隐私机制，对上传原型或梯度施加噪声，以缓解推理与恶意攻击带来的风险；在带宽受限条件下，仍能通过语义感知压缩保障多轮联邦迭代的可行性与通信效率。

上述业务功能需求的关系与数据流如图~\ref{fig:sys_arch} 所示；实现层面的技术组件依赖关系见图~\ref{fig:sys_tech_flow}（均见 5.2.1 节）。

\subsubsection{用户需求分析}

系统主要面向学术研究人员，功能上包括：个人画像管理，供用户维护研究兴趣、机构与方向等属性作为冷启动属性视图输入，数据仅存本地并以加密形式参与训练；联邦训练监控，为管理员提供 Loss、Hit@5、NDCG@5 等曲线与在线状态，支撑异常诊断与调度；学术资源推荐，依据是否有交互历史在“结构优先”与“属性优先”策略间切换，给出个性化论文列表并可结合元路径注意力做可解释展示。

系统用户需求与上述模块的对应关系已在 5.3.3 与 5.4 节的实现与测试中体现。

\subsection{系统总体架构设计}
\label{sec:sys_architecture}

本系统采用分层架构设计，实现了算法逻辑与业务应用的高度解耦。

\subsubsection{逻辑架构设计}

系统逻辑上自下而上划分为四层，如图~\ref{fig:sys_arch} 所示。底层为数据存储层：客户端以 SQLite 保存自我中心子图与用户属性，服务端以 MySQL 与 Redis 维护全局模型参数与联邦任务元数据，原始交互保留在端侧而不出域，端云仅交换经保护的参数、梯度或语义原型。其上是算法引擎层，封装 FedASCL 的双视图对比与原型对齐、第四章语义感知压缩（动态元路径选择器与残差梯度量化），并由服务器对客户端上传更新执行加权聚合。业务服务层提供推荐列表、冷启动推断与异步训练调度等 API；顶层业务展示层基于 React 实现，支持学术文献浏览与联邦训练状态可视化。

\subsubsection{联邦协同训练模块实现}

该模块协调分布式客户端与中心服务器进行多轮联邦学习，实现模型分发（广播）、本地更新、压缩与上传、安全聚合四个阶段的闭环。
\begin{cnenumerate}
    \cnitem 同步联邦训练流程。单轮训练包含以下步骤，即模型分发（广播）——服务端随机选择 $K$ 个客户端，下发全局模型参数 $W_g$ 与全局属性原型 $P_g$；本地更新——客户端基于本地异构子图执行多轮 FedASCL 训练，双视图编码器分别提取结构特征与属性特征，通过最大化互信息优化节点表示；压缩与上传——训练结束后，客户端经语义感知压缩器处理梯度，仅上传压缩后的梯度更新与本地聚类中心，原始图数据保留在本地；安全聚合——服务端收集各节点更新，执行加权聚合，更新全局模型版本。
    \cnitem 仿真引擎与分级推荐路由。系统根据用户是否具备交互历史，在常规推荐（结构视图单通路）与冷启动推荐（FedASCL 属性—结构双视图）之间进行路由，总体流程如图~\ref{fig:simulator_arch} 所示。工程上，交互历史充分的用户优先采用与 FedHGNN 同类异构图结构编码一致的轻量前向路径，以降低在线推断时双编码与对比模块的算力开销；该路由属于在线推断期的算力折中，联邦训练与全局收敛仍以 FedASCL 双视图及原型对齐为主，与第三章实验设定一致；冷启动推荐效果以第三章表~\ref{tab:cold_start_res} 的离线评测为准，不宜将结构优先路径理解为额外一组算法对比实验。为验证算法可行性，后端同时实现高保真仿真引擎：\texttt{Data\allowbreak Partitioner} 将 ACM 等数据集切分为多个互不重叠的 Ego-Subgraph，模拟 Non-IID 分布；多线程机制模拟客户端并发训练与网络延迟。
\end{cnenumerate}

\subsubsection{学术资源推荐与冷启动模块实现}

该模块基于训练好的联邦异构图模型为用户提供个性化学术资源推荐，并重点支持冷启动场景。系统按交互历史路由不同推理分支（图~\ref{fig:rec_logic}）：活跃用户走结构优先路径，聚合“用户-论文-会议”等高阶邻域生成嵌入并检索 Top-N 文献；新用户无交互时启用 FedASCL 属性分支，将兴趣、机构等映射至全局原型空间再做相似检索，实现零样本冷启动。需指出：训练阶段各客户端仍统一执行 FedASCL 联合优化，与第三章一致；结构优先路径仅用于交互充足用户的在线推断，与冷启动离线实验的设定分工不同。同时以元路径注意力权重增强可解释性，例如 \texttt{User-\allowbreak Author-\allowbreak User} 权重高时，可提示“推荐该学者是因为你们有共同的合作者”。

\subsubsection{通信压缩与隐私模块实现}

客户端上传阶段集成第四章的语义感知压缩（动态元路径选择器与残差梯度量化），逻辑同 4.2--4.3 节，在约 $20\times$ 压缩比下兼顾精度与通信开销；同时对上传的全局原型或梯度施加拉普拉斯噪声（Laplace Mechanism），通过配置隐私预算 $\varepsilon$ 与敏感度，在 $\varepsilon$-差分隐私（$\delta=0$）意义上抑制标签推理与后门风险，并尽量控制对收敛的影响。

\subsubsection{性能与鲁棒性验证}

除上述界面层验证外，以下从通信与挂钟时间补充系统级性能数据。

通信开销对比。在相同数据集与训练轮次下，对比语义感知压缩开关时的累计流量；仿真采用第四章一致配置（$\rho=0.2$、$b=4$，约 $20\times$ 压缩比），100 轮、每轮 10 客户端场景下，全精度上传累计约 12.2 MB，开启压缩后约 0.62 MB，压缩比约 $20\times$，与表~\ref{tab:communication_cost} 量级一致；挂钟时间由约 45 分钟降至约 6 分钟，加速约 $7.5\times$，表明集成后通信开销可降低约一个数量级，与第四章结论一致。

测试结果表明，系统在仿真环境下实现了模型分发、本地训练、梯度聚合及在线推荐等流程闭环；在隐私保护与通信效率方面，数据隔离与语义感知压缩达到设计目标，验证了 FedASCL 与压缩策略集成的工程可行性。推荐排序的定量结论仍以第三章离线实验为准。

\subsection{本章小结}
\label{sec:sys_summary}

本章阐述了基于 FedASCL 与语义感知压缩策略的联邦论文推荐原型系统的设计与实现。在需求分析部分，明确了联邦身份认证、端云协同训练、隐私保障与高效通信等业务需求，以及个人画像管理、训练监控与学术推荐等用户需求。在架构设计部分，给出了自下而上的四层逻辑架构与数据不出域机制。在关键实现部分，从开发环境、联邦协同训练、学术资源推荐与冷启动、通信压缩与隐私等模块做了介绍。功能测试与通信仿真表明，系统在模块集成、数据隔离与通信压缩方面达到设计目标；算法层面的排序效果以第三章离线实验为准，本章完成的是算法到原型的工程验证而非生产环境部署。
