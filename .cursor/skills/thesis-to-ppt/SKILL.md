# 论文 → 答辩 / 汇报 PPT（Skill）

## 何时启用

用户希望**根据当前仓库中的学位论文 LaTeX 源码**生成答辩、开题复盘或精简汇报用的 **PPT 内容**（大纲、逐页讲稿、或 Marp / 可导入 Office 的结构化文本）。

## 输入与优先读取顺序

1. `main.tex` / `main_inner.tex`：确认章节 `\input` 链路与章标题。
2. `chapters/chap1_intro.tex` … `chapters/chap6_summary.tex`（及 `abstract.tex`）：抽取各章**核心问题、方法名、公式仅保留符号级一句、实验表/图编号与结论句**。
3. 若存在 `评阅意见与正文修改对照.md`：可生成**1～2 页「创新点与边界」备用页**（不展开评阅字样，只写学术表述）。

**不要**把 PDF 二进制当正文解析；以 `.tex` 为准。若用户只给 PDF，说明局限并尽量让其提供对应 `.tex` 或章名。

## 输出形态（由用户一句话选定；未指定时默认 A）

| 选项 | 输出 | 适用 |
|------|------|------|
| **A** | **Marp** 兼容 Markdown（`---` 分页、`#` 标题、条列、备注用 HTML 注释） | 可导出 PDF/PPTX，适合技术用户 |
| **B** | **分节大纲 + 每页条目标题与 3～5 条要点**（纯 Markdown） | 用户自行粘到 WPS/Office |
| **C** | **逐页讲稿**（同结构，每页下附「口播 30～60 秒」） | 背稿用 |

## 幻灯片逻辑结构（硕士答辩常用）

按顺序组织章节页，**控制总页数**（建议陈述主体 15～22 页 + 致谢/附录可选）：

1. **封面**：题目、姓名、学号、导师、学校、日期（占位符保留）。
2. **目录**：与论文章节对齐（绪论 / 相关… / 方法 / 压缩 / 系统 / 总结）。
3. **研究背景与意义**（来自第一章）：问题三要素（冷启动、Non-IID、通信）各 1 条，**不堆文献**。
4. **国内外现状**（第一章 1.2 压缩）：每块 1～2 页，结尾用「不足 → 本文切入点」收束。
5. **本文工作与技术路线**（贡献节 + 第三章入口）：**一张总框图**占位（说明用论文哪张图 `fig:fedascl_framework`）。
6. **FedASCL**（第三章）：问题说明 → 双视图与对比 → 原型 → 元路径与损失；公式**只保留名称**，避免整页公式。
7. **实验**（第三章）：RQ1/2/3 各 1 页，**表号图号与论文一致**，结论只写相对趋势与关键数字（与正文一致，不编造）。
8. **压缩策略**（第四章）：动机 → 剪枝+量化 → 关键表/通信量；**若正文未含 ACM 压缩数，幻灯片亦不虚构**，可用「两数据集 + 原因一句」与正文对齐。
9. **系统**（第五章）：需求 → 逻辑架构 `fig:sys_arch` + 技术栈 `fig:sys_tech_flow` → 测试结论一句。
10. **总结与展望**（第六章）：三点贡献 + 展望一条即可。
11. **谢谢 / Q&A**（可选备用：创新点、局限、与基线差异三句）。

## 版式与语言

- 语言：**简体中文**（与论文一致）。
- 条列：**短句**，每页不超过 6 条；关键词可加粗（Marp 用 `**`）。
- **图**：用占位 `[图：论文图 X]` 并写明 `\ref` 或文件名，由用户从论文 PDF 截屏替换。
- **色与模板**：不强制品牌色；若用户指定学校模板，按其要求改 `style` 或文字说明。

## 工作流（Agent 执行顺序）

1. glob / read `main_inner.tex` 或 `main.tex`，列出 `chapters/*.tex`。
2. 每章抓：**节标题 1 级列表** + **每节最多 2 句可念结论** + **表/图标签**（grep `\label{fig:` `\label{tab:`）。
3. 按上文「幻灯片逻辑结构」生成选定格式输出。
4. 文末附 **「答辩可能被问」3 条** + **建议翻论文 PDF 的页码/标签**（若能从结构推断则写「第三章实验节」等，勿瞎编页码）。

## 与「生成整份 .pptx」的边界

- Agent **默认输出 Markdown / 大纲**；若用户明确要求 **python-pptx** 或可运行脚本，再在其环境中执行并说明依赖 `pip install python-pptx`。
- 不替用户运行需授权的商业模板网站。
